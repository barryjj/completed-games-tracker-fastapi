{# The three filter selects for import review, in one place so the inline copy (rendered inside the filter form on full page load) and the OOB copy (re-emitted on a tab switch, when `oob` is true) can never drift apart. Each select carries its OWN hx-get (default trigger for a select is `change`) rather than the form listening via `change from:#…`. That form listener bound to the ORIGINAL element at load time, so when a tab switch replaced the select out-of-band the new element had no listener and the filter silently stopped working. Self-contained triggers get re-bound by htmx every time it processes the swapped-in select, so OOB replacement is safe. hx-include pulls the sibling fields (tab/view/q + the other selects) so a change sends the full filter state. #}
